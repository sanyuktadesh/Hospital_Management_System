# SOFTWARE REQUIREMENTS SPECIFICATION
# HOSPITAL MANAGEMENT SYSTEM
# Version 4.0

## 1. INTRODUCTION
### 1.1 Purpose
This document describes the requirements for the Hospital Management System.

## 2. FUNCTIONAL REQUIREMENTS

### 2.1 Patient Management Module
- Register new patients
- Update patient details
- View patient history

### 2.2 Doctor Management Module
- Register doctors
- Update doctor profiles
- View specializations

### 2.3 Appointment Management Module
- Book appointments
- Cancel appointments
- View doctor availability

### 2.4 Billing Module
- Generate bills
- Process payments
- Store receipts

### 2.5 Medical Records Management Module
- Store patient diagnosis
- Store prescriptions
- Upload medical reports
- View past treatment history

### 2.6 Staff Management Module
- Register hospital staff
- Update staff details
- Assign roles and duties
- View staff schedules

### 2.7 Inventory Management Module
- Track medicines and supplies
- Update stock levels
- Set reorder alerts
- Generate stock reports

### 2.8 Room & Bed Allocation Module
- Assign rooms and beds to patients
- Track occupied/available beds
- Update discharge details

### 2.9 Emergency Management Module
- Register emergency cases
- Prioritize critical patients
- Notify available doctors


## 3. NON-FUNCTIONAL REQUIREMENTS

### 3.1 Reliability
System should work without failure and ensure accuracy of data.

### 3.2 Security
Access must be restricted based on user roles. Sensitive data must be protected.

### 3.3 Usability
Interface must be simple for staff with minimal training.

### 3.4 Performance
System should quickly retrieve and update patient and doctor records.

### 3.5 Scalability
System should handle increased number of patients, doctors, and records.


## 4. CONCLUSION
This version includes extended modules for staff, inventory, emergency handling,
and room allocation to make the Hospital Management System more complete.
